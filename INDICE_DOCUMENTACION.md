## 💡 CONCEPTOS CLAVE

### Verificación de Permisos
```
¿Usuario tiene permiso? → SÍ → Permitir acción → Guardar en DB
                       → NO → Bloqueado → Mostrar error
```

### Roles
- **Admin**: Acceso total al sistema
- **Cliente**: Acceso limitado a sus datos

### Permisos
- Más de 30 permisos definidos
- Cada rol tiene su conjunto de permisos
- Verificación en CADA acción crítica

### Protecciones
1. Historial contable protegido
2. Validación de stock
3. Seguridad de contraseñas
4. Restricción de datos
